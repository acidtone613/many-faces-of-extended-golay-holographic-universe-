3.1 H1 — The Logical Symplectic Matrix of U
─────────────────────────────────────────────
The operator U = H_L · P23 induces a logical gate on the 12 logical qubits.
Its symplectic matrix M_U ∈ Sp(12, 𝔽₂) was computed using the RC-133b logical
basis:

  X̄_j = (c_j, 0)    where c_j = (e_j B⁻¹, e_j) ∈ C
  Z̄_j = (f_j, f_j)  where f_j is the j-th free column unit vector

Verification: M_U^T Ω M_U = Ω (mod 2) — PASS.

All five engine generators (P23, P11, S, H_L, U) pass the symplectic check
in the logical basis.

3.3 H3 — The Full Engine Generates the Logical Clifford Group
──────────────────────────────────────────────────────────────
Group generated by ⟨P23, P11, S, H_L⟩ in the logical basis:
• Size: 6,072 = 2³ × 3 × 11 × 23
• Element orders: {1, 2, 3, 4, 6, 11, 22, 23, 46, ...}
• Classical-type gates (B=C=0): 6,072/6,072 (100%)

The group is large and non-trivial but consists entirely of classical-type

Verdict: PASS (large subgroup generated, though not full Clifford).

3.4 H4 — U_θ is a Non-Clifford Phase Gate
───────────────────────────────────────────
U_θ is a physical rotation in the (22,23) coordinate plane with angle θ = π/23.
• Order: 46 (since 46 × π/23 = 2π)
• Phase: π/23
• Non-Clifford: TRUE (Clifford phase gates have orders 4 or 8)

Note: U_θ is a continuous unitary on the Hilbert space, not a symplectic
matrix over 𝔽₂. It does not preserve the stabilizer group and thus cannot be
used as a logical gate within the stabilizer formalism.

Verdict: PASS (by construction, U_θ is non-Clifford).

================================================================================
4. ROOT CAUSE ANALYSIS
================================================================================

The failure of C5 (entanglement) is traced to a basis choice artifact:

| Basis Property | Consequence |
|:---------------|:------------|
| X̄_j = (c_j, 0) — pure X-type | H_L swaps X↔Z, giving (0, c_j) |
| Z̄_j = (f_j, f_j) — Y-type | H_L fixes Z̄_j (symmetric under swap) |
| Result | H_L maps X-operators to X-operators, not X↔Z swap |

In this basis, H_L does not act as a logical Hadamard. It acts as a classical
operation that permutes X-operators while fixing Z-operators. Consequently,
all generated gates are block-diagonal with B=C=0.

This is a basis artifact, not a structural failure of the framework. A
different choice of logical basis (e.g., X̄_j = (c_j, 0), Z̄_j = (0, f_j)) might
make H_L act as a genuine Hadamard and reveal entangling structure.

================================================================================
5. HONEST LIMITATIONS
================================================================================

1. BASIS DEPENDENCE
   The logical symplectic matrix depends on the choice of logical basis.
   The RC-133b basis (free columns 12–23) was used. A different basis may
   yield a different M_U with entangling blocks.

2. U_θ IS NOT A LOGICAL GATE
   U_θ is a physical rotation, not a symplectic logical operator. It does not
   preserve the stabilizer group. Its "logical" action is only meaningful in
   the broader Hilbert space, not within the stabilizer formalism.

3. GROUP SIZE VS. STRUCTURE
   The generated group has 6,072 elements — large but entirely classical-type.
   This suggests the engine's non-abelian structure (C₂₃ ⋊ C₁₁) manifests as
   permutations and phases, not as genuine quantum entanglement in this basis.

4. COMPUTATIONAL LIMITATIONS
   The full Clifford group on 12 qubits has ~2¹⁴⁴ elements. We tested subgroup
   generation up to 50,000 elements. The group may grow larger with more
   generators or a different basis.

All computations were performed in Python 3 with NumPy.
The complete reproduction script is provided as a companion file:
  • RC-146_reproduction.py

Key parameters:
  • Generator polynomial: g(x) = x¹¹ + x¹⁰ + x⁶ + x⁵ + x⁴ + x² + 1
  • P23: cyclic shift on positions 0–22, position 23 fixed
  • P11: multiplication by 2 mod 23 on positions 0–22, position 23 fixed
  • S: x → -x⁻¹ mod 23 for 1≤x≤22, S(0)=23, S(23)=0
  • H_L: swap X and Z parts (48D symplectic matrix)
  • Logical basis: X̄_j = (c_j, 0), Z̄_j = (f_j, f_j) with c_j = (e_j B⁻¹, e_j)
